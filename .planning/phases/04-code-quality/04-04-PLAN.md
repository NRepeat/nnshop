<objective>
Replace bare `setTimeout(...)` calls in four components with the `useRef` pattern so timer IDs are captured and cleared on unmount. This prevents stale callbacks from firing after components unmount (route changes, modal closes, etc.).

Purpose: Uncaptured setTimeout IDs cause React state updates on unmounted components — a source of memory leaks and spurious errors. The fix is mechanical: one ref per timer, one clearTimeout in cleanup.
Output: Four files updated with the standard ref pattern; no runtime behavior changed.
</objective>
