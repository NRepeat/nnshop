<objective>
Replace cookie-based gender state with URL-derived gender to fix back-navigation bugs where the UI would show stale gender context.
</objective>

<tasks>

  <name>Task 1: Refactor Middleware and Global Utilities</name>
  <files>proxy.ts, src/features/header/navigation/api/setCookieGender.ts</files>
  <action>
    - Refactor `proxy.ts` to remove all `response.cookies.set('gender', ...)` and `request.cookies.get('gender')` calls.
    - Rely purely on URL segments for gender detection.
    - Keep forwarding `x-gender` header as a fallback for Server Components that cannot easily access `params`.
    - Clean up `src/features/header/navigation/api/setCookieGender.ts` (remove cookie operations).
  </action>
  <verify>
    <automated>grep -r "gender" proxy.ts | grep "cookie" || true</automated>

  </verify>
  <done>Middleware no longer interacts with gender cookies.</done>
