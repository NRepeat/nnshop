**Step 1 — Create the utility:**

Create `src/shared/lib/utils/getCurrencySymbol.ts`:
```typescript
import getSymbolFromCurrency from 'currency-symbol-map';

/**
 * Returns the display label for a given ISO currency code.
 * UAH is overridden to return 'грн' instead of the default '₴' symbol.
 */
export function getCurrencySymbol(currencyCode: string): string {
  if (currencyCode === 'UAH') return 'грн';
  return getSymbolFromCurrency(currencyCode) || currencyCode;
}
```

**Step 2 — Update each of the 11 price files:**

Read each file first, then apply these changes:
- Add import: `import { getCurrencySymbol } from '@shared/lib/utils/getCurrencySymbol';`
- Replace the pattern:
  ```typescript
  // BEFORE:
  import getSymbolFromCurrency from 'currency-symbol-map';
  // ...
  const currency = getSymbolFromCurrency(someObj.currencyCode) || someObj.currencyCode;
  ```
  With:
  ```typescript
  // AFTER:
  import { getCurrencySymbol } from '@shared/lib/utils/getCurrencySymbol';
  // ...
  const currency = getCurrencySymbol(someObj.currencyCode);
  ```
- Remove the `import getSymbolFromCurrency from 'currency-symbol-map'` line (no longer needed in the file)

The variable name `currency` and its usage in JSX remain unchanged — only the import and the assignment line change.

Some files may use `currencyCode` from different source objects (e.g., `item.cost.totalAmount.currencyCode`, `product.priceRange.minVariantPrice.currencyCode`). Read each file to confirm the exact source, but the replacement pattern is always the same: replace the `getSymbolFromCurrency(x) || x` call with `getCurrencySymbol(x)`.
