<objective>
Fix two SEO technical bugs where Next.js streaming metadata causes `<title>` and `<link rel="canonical">` to appear in `<body>` instead of `<head>` on pages with async generateMetadata.

Root cause: Next.js 15.2+ introduced streaming metadata — async generateMetadata functions are deferred and injected after initial HTML streaming, landing outside `<head>` for non-bot user agents (including Screaming Frog).

Fix: Set `htmlLimitedBots: /.*/` in next.config.ts to force all requests (not just known bots) to wait for metadata resolution before streaming begins. Also move the misplaced `<Suspense><DraftModeTools /></Suspense>` from after `</body>` to inside `<body>` — it currently produces invalid HTML structure.

Purpose: Ensure Screaming Frog, Google, and all crawlers reliably find `<title>` and canonical tags inside `<head>`.
Output: next.config.ts with htmlLimitedBots, layout.tsx with valid HTML structure.
</objective>
