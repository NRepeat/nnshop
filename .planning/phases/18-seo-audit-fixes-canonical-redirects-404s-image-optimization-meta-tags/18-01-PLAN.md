# Phase 18: SEO Audit Fixes (Canonical, Redirects, 404s, Image Optimization, Meta-Tags)

## Objective
Remediate remaining SEO audit issues identified in the March 2026 full audit. This includes technical fixes for canonical tags, redirect logic, 404 error handling, and performance-related image optimizations.

## Key Files & Context
- `proxy.ts`: Root middleware handling normalization, redirects, and route guarding.
- `app/[locale]/(frontend)/search/page.tsx`: Search results page.
- `app/robots.ts`: Crawler instructions.
- `src/entities/product/ui/ProductCard.tsx`: Product display and hover preloading logic.
- `lib/imageLoader.ts`: Global image optimization settings.

## Implementation Steps

### 1. Middleware (`proxy.ts`) Optimization & Fixes
- **Fix Locale Redirect Bug**: Resolve the `/ru` -> `/uk/woman` redirect issue identified in tests (it should redirect to `/ru/woman`).
- **Search Query Normalization**: Implement lowercase normalization for the `q` search parameter to prevent duplicate content indexing and consolidate crawl budget.
- **Goal State Refinement**: Ensure all redirects are single-hop 301s to the final canonical URL.

### 2. Robots.txt & Sitemap Refinement
- **Update Robots.txt**: Consolidate search disallow rules and ensure `/*?q=` is correctly blocking all search variations.
- **Sitemap Verification**: Ensure dynamically generated search pages (even with `noindex`) are not present in any sitemap files.

### 3. Meta-Tag & Canonical Adjustments
- **Title/Description Length Fixes**: Identify specific pages with titles < 30 or > 60 chars and descriptions < 70 chars. Apply manual overrides where the generic templates fall short.
- **Canonical Status on 404s**: Ensure 404 pages (like those triggered by repetitive paths) return a true 404 status and don't contain self-referencing canonical tags.

### 4. Image Optimization & Performance
- **Hover Preload Reduction**: Modify `ProductCard.tsx` to only preload the first 1-2 images on hover instead of 5, reducing unnecessary bandwidth and improving initial page performance.
- **Quality Tuning**: Adjust default quality in `imageLoader.ts` from 65 to 60.

### 5. Validation & Testing
- Update `src/shared/lib/tests/proxy.test.ts` to include the search normalization and the fixed locale redirect.
- Run all SEO-related tests and verify production headers.
