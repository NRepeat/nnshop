<objective>
Fix invalid HTML structure: `<li>` elements without `<ul>` parents in navigation, and `<h3>` accordion headers breaking heading sequence.

Purpose: Lighthouse flags two HTML structure violations. First, `PersistLinkNavigation.tsx` renders `NavigationMenuItem` components (Radix `Primitive.li` elements) inside a plain `<div>` with no `<ul>` parent — this is invalid HTML (A11Y-03). Second, `AccordionPrimitive.Header` in `accordion.tsx` renders as `<h3>` by default; on the product page this creates an h1 → h3 heading sequence skipping h2 (A11Y-03). After these fixes, a build verification gate confirms the entire Phase 7 set of changes compiles and functions correctly.

Output: Updated accordion.tsx (div header via asChild), updated PersistLinkNavigation.tsx (plain ul/li), and a human-verify checkpoint confirming no browser errors on a product page.
</objective>
