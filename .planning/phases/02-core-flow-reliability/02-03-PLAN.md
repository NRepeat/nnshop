<objective>
Fix account linking to surface partial failures from Promise.allSettled and propagate Prisma transaction errors.

Purpose: Two related bugs. First (auth.ts): `Promise.allSettled([...])` result is discarded — failures are invisible even if cart merge or data link fail. Second (on-link-account.ts): `linkAnonymousDataToUser` catches Prisma errors internally without re-throwing, so `Promise.allSettled` always sees a fulfilled promise even when the data link silently failed. Per CONTEXT.md decision: "If a Prisma transaction fails: propagate the error up." These fixes make failures detectable and observable.

Output: `auth.ts` captures the `allSettled` result in named variables and logs rejected outcomes; `on-link-account.ts` re-throws after logging so the rejection surfaces to the `allSettled` caller.
</objective>

} catch (error) {

    step: 'prisma-link-anonymous-data',
    error: error instanceof Error ? error.message : String(error),

Change to capture results and check each by name. Remove the outer try/catch (Promise.allSettled never rejects — it always fulfills with an array of results, so the try/catch is not needed and is misleading):

```typescript
onLinkAccount: async ({ anonymousUser, newUser }) => {
  const [cartMergeResult, dataLinkResult] = await Promise.allSettled([
    anonymousCartBuyerIdentityUpdate({ anonymousUser, newUser }),
    linkAnonymousDataToUser({
      anonymousUserId: anonymousUser.user.id,
      newUserId: newUser.user.id,
    }),
  ]);

  if (cartMergeResult.status === 'rejected') {
    console.error('[onLinkAccount] cart merge failed', {
      step: 'anonymous-cart-buyer-identity-update',
      userId: newUser.user.id,
      orderId: undefined,
      error:
        cartMergeResult.reason instanceof Error
          ? cartMergeResult.reason.message
          : String(cartMergeResult.reason),
    });
  }

  if (dataLinkResult.status === 'rejected') {
    console.error('[onLinkAccount] data link failed', {
      step: 'link-anonymous-data-to-user',
      userId: newUser.user.id,
      orderId: undefined,
      error:
        dataLinkResult.reason instanceof Error
          ? dataLinkResult.reason.message
          : String(dataLinkResult.reason),
    });
  }
},
```

The outer try/catch is removed because `Promise.allSettled` never rejects — wrapping it in try/catch creates a false impression that it can throw, and the inner errors are handled per-result above.

TypeScript note: `cartMergeResult.reason` is typed as `any` in the PromiseSettledResult type — the `instanceof Error` check is correct. No type assertions needed.
