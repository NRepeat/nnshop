<objective>
Make order creation resilient to partial failures and null cart delivery state.

Purpose: Two distinct bugs live in `create.ts`. First (BUG-05): `cart.delivery.addresses.find(...)` at line 208 throws when `cart.delivery` is null — which happens for a cart that was just created or is mid-merge. Second (RELY-02): if the Shopify Admin API order creation succeeds but the subsequent `prisma.order.create` throws, the outer catch returns `{ success: false }` — the user sees an error even though their order exists in Shopify. Both fixes are surgical changes to the same file.

Output: A patched `create.ts` where `cart.delivery` is accessed via optional chaining, and the Prisma DB save is wrapped in an isolated try/catch that logs on failure and returns success regardless.
</objective>

<task type="auto">
  <name>Task 2: Isolate DB save failure from Shopify order success (RELY-02)</name>
  <files>src/features/order/api/create.ts</files>
  <action>
In `create.ts`, find the block after `const createdOrder = orderResponse.orderCreate.order;` succeeds (after the `userErrors` check and the `!createdOrder` guard). Currently the code is:

```typescript
// Delete any existing draft orders for this user
await prisma.order.deleteMany({
  where: { userId: session.user.id, draft: true },
});

await prisma.order.create({
  data: {
    shopifyOrderId: createdOrder.id,
    orderName: createdOrder.name,
    userId: session.user.id,
    draft: false,
  },
});

return {
  success: true,
  order: createdOrder,
};
```

Wrap BOTH the `prisma.order.deleteMany` and `prisma.order.create` calls in a single isolated try/catch. The catch block must:
1. Log with the required structured shape (for Phase 5 Sentry upgrade compatibility)
2. NOT re-throw — Shopify order already exists, user must see success

```typescript
// DB save is best-effort — Shopify order already exists and is confirmed
try {
  await prisma.order.deleteMany({
    where: { userId: session.user.id, draft: true },
  });
  await prisma.order.create({
    data: {
      shopifyOrderId: createdOrder.id,
      orderName: createdOrder.name,
      userId: session.user.id,
      draft: false,
    },
  });
} catch (dbError) {
  console.error('[create-order] DB save failed after Shopify success', {
    step: 'prisma-order-create',
    userId: session.user.id,
    orderId: createdOrder.id,
    error: dbError instanceof Error ? dbError.message : String(dbError),
  });
  // Do NOT re-throw — Shopify order exists; user must see success state
}

return {
  success: true,
  order: createdOrder,
};
```

The outer `catch (error)` block remains unchanged — it handles failures that occur before Shopify order creation (session missing, cart missing, API call failure). Only the DB save needs the isolated catch.
